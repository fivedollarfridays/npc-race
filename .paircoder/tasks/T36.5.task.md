# T36.5 — Update CI Workflow

Restructure `.github/workflows/ci.yml` for 3-tier test execution.

## New CI jobs

### 1. lint (unchanged)
### 2. validate-cars (unchanged)

### 3. test-unit (new) — runs on every push
```yaml
- name: Run unit tests
  run: pytest -m "not smoke and not integration" --tb=short -q
  timeout-minutes: 3
```

### 4. test-smoke (new) — runs on every push
```yaml
- name: Run smoke tests
  run: pytest -m smoke --tb=short -q
  timeout-minutes: 5
```

### 5. test-integration — runs on PR with label OR manual trigger
```yaml
if: github.event_name == 'pull_request' && contains(github.event.pull_request.labels.*.name, 'ready-to-merge')
- name: Run integration tests
  run: pytest -m integration --tb=short -q
  timeout-minutes: 15
```

## Target CI times
- lint + validate + unit: < 2 min
- smoke: < 3 min (parallel with unit)
- Total on push: < 3 min
- Integration (on demand): < 15 min

## AC
- [ ] CI passes on push in < 3 min
- [ ] Unit + smoke run on every push
- [ ] Integration only runs on labeled PRs
- [ ] All 3 tiers produce separate job results
