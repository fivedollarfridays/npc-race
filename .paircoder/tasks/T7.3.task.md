# T7.3 — Tire Model Upgrade: Quadratic Curves

## Objective

Upgrade two aspects of the tire model based on research findings:

1. **Pre-cliff degradation**: Change from linear to quadratic (convex curve). Research universally agrees: fresh tires degrade slowly, worn tires degrade faster — even before the cliff.

2. **Temperature-grip curve**: Change from piecewise linear to smooth quadratic parabola (Speed Dreams model). Produces more realistic smooth grip rolloff outside the optimal window.

## Research Basis

- All academic models (TUMFTM, f1metrics, Speed Dreams) agree: degradation is convex
- Speed Dreams: `mu(T) = mu0 * max(1 - k*(T-Topt)², 0.1)` — smooth parabola
- Current linear pre-cliff: `grip = base_grip * (1.0 - wear * 0.3)` — wrong shape

## Implementation Plan

### Update `engine/tire_model.py`

Change the pre-cliff grip formula:

```python
# Before (linear):
grip = base_grip * (1.0 - wear * 0.3)

# After (quadratic — wear^1.5 makes degradation accelerate):
grip = base_grip * (1.0 - wear ** 1.5 * 0.3)
```

This means:
- At wear=0.2: old=0.94, new=0.973 (less degraded — fresh tires last better)
- At wear=0.5: old=0.85, new=0.894 (still close)
- At wear=0.7: old=0.79, new=0.824 (slight improvement)
- At wear=0.8 (near cliff): old=0.76, new=0.785 (converging)

The quadratic form preserves early-stint freshness while accelerating late-stint drop.

### Update `engine/tire_temperature.py`

Replace the piecewise linear grip curve with quadratic parabola:

```python
# Before (piecewise linear with 3 zones):
if tire_temp < low_edge:
    t = (tire_temp - AMBIENT_TEMP) / (low_edge - AMBIENT_TEMP)
    return 0.5 + 0.5 * t
elif tire_temp > high_edge:
    ...
else:
    return 1.0

# After (smooth quadratic — single formula):
def tire_temp_grip_factor(tire_temp: float, compound: str) -> float:
    optimal = OPTIMAL_TEMP[compound]
    window = TEMP_WINDOW[compound]
    # k calibrated so grip = 0.5 at extremes (AMBIENT or 150°C)
    k = 0.5 / max(1.0, (optimal - AMBIENT_TEMP)) ** 2
    factor = 1.0 - k * (tire_temp - optimal) ** 2
    return max(0.5, min(1.0, factor))
```

This produces a smooth parabola centered on optimal temp:
- At optimal: 1.0 (peak grip)
- Below optimal: smooth decrease toward 0.5
- Above optimal: smooth decrease toward 0.5
- Always clamped to [0.5, 1.0]

### Friction-power-based heat (optional improvement)

If line budget allows, upgrade heat generation to be proportional to sliding energy:

```python
# Before: throttle * 2.0 + curvature * 8.0 + abs(lateral) * 1.5
# After: add grip-demand-based heat
grip_demand = curvature * speed / 100.0  # higher speed corners = more energy
heat = (throttle * 2.0 + grip_demand * 5.0 + abs(lateral) * 2.0) * dt
```

This ties heat to actual cornering energy (speed × curvature) rather than just curvature alone.
