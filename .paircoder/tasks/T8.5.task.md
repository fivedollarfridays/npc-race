# T8.5 — Telemetry Strip: Time-Series Charts

## Objective

Build the bottom strip: 3 stacked canvas-based time-series charts showing the selected car's performance over a rolling 3-lap window. Speed trace (primary), tire wear+temp (secondary), gap-to-leader (tertiary).

## New File: `viewer/js/telemetry-strip.js` (~200 LOC)

### Three chart rows

**Row 1: Speed trace** (height: 80px)
- X-axis: distance around track (or elapsed ticks for last 3 laps)
- Y-axis: 0-370 km/h
- White line: current speed
- Gray filled area underneath showing track curvature (from `track_curvatures`)
- Vertical dashed lines at sector boundaries
- Event markers: ● pit stops, ▲ overtakes

**Row 2: Tire wear + temp** (height: 50px)
- Green line: tire wear (0-1 scale, left Y-axis)
- Orange line: tire temp (right Y-axis, 20-150°C)
- Red dashed line: cliff threshold (0.75-0.80 depending on compound)
- Compound color background bands (soft stint = red tint, medium = yellow tint)

**Row 3: Gap to leader** (height: 50px)
- White line: gap_ahead_s over time
- Green fill when gap is shrinking (good)
- Red fill when gap is growing (bad)
- Pit stop gaps show as vertical jumps (expected — don't alarm-color them)

### Rolling window

Show the last N ticks where N = 3 laps worth of ticks (~3 × 56s × 30 tps = 5040 ticks). As race progresses, window scrolls right.

At race end (diagnostic mode toggle), switch to showing ALL ticks.

### Data buffering

Client-side: maintain a circular buffer per car for the selected car's history:
```javascript
const carHistory = {
  speed: [],      // last N frames of speed values
  tire_wear: [],
  tire_temp: [],
  gap_ahead: [],
  events: [],     // {tick, type, label}
};
```

Populate from replay frames as they play. When car is changed, rebuild from available frame data.

### Chart rendering

Use `<canvas>` 2D context. No external libraries. Simple line charts:

```javascript
function drawLineChart(ctx, data, yMin, yMax, color, width, height) {
  ctx.beginPath();
  for (let i = 0; i < data.length; i++) {
    const x = (i / data.length) * width;
    const y = height - ((data[i] - yMin) / (yMax - yMin)) * height;
    if (i === 0) ctx.moveTo(x, y);
    else ctx.lineTo(x, y);
  }
  ctx.strokeStyle = color;
  ctx.lineWidth = 1.5;
  ctx.stroke();
}
```

### Performance

Only redraw every 3rd frame (10 fps for charts is fine, saves CPU). The race runs at 30 tps but chart updates at 10 fps.
