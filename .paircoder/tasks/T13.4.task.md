# T13.4 — Wire into Race Runner + Replay Export

## Objective

Wire the narrative engine into `engine/race_runner.py` so that after each race:
1. Events are detected from the replay
2. Commentary is generated
3. Race report is generated
4. All are included in the replay JSON export

## Implementation Plan

### In race_runner.py

After `sim.run()` and before saving replay JSON:
```python
from .narrative import detect_events
from .commentary import format_events
from .race_report import generate_report

events = detect_events(replay_data["frames"], replay_data.get("ticks_per_sec", 30))
commentary = format_events(events)
report = generate_report(results, events, commentary, track_name=track_name)
replay_data["events"] = [{"type": e.type, "tick": e.tick, "cars": e.cars, "data": e.data} for e in events]
replay_data["commentary"] = commentary
replay_data["race_report"] = report
```

### In engine/__init__.py

Add exports for narrative, commentary, race_report modules.

### Console output

Print the race report to console after results (optional, behind a flag or always).

## Tests

- `test_replay_has_events` — replay JSON has "events" list
- `test_replay_has_commentary` — replay JSON has "commentary" list
- `test_replay_has_race_report` — replay JSON has "race_report" string
- `test_race_report_printed` — (optional) verify console output

## Acceptance Criteria

- race_runner.py wires narrative pipeline after race
- Replay JSON includes events, commentary, race_report
- No simulation.py changes
- All tests pass, ruff clean
