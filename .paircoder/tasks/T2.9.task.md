# Objective

Add three camera modes: full track (default), follow cam (tracks selected car), and onboard cam (tight zoom with rotation). Smooth transitions between modes.

# Implementation Plan

1. Create `viewer/js/camera.js` with `CameraSystem` class
2. Full track view (default): current behavior — fit entire track in viewport
3. Follow cam: zoom to ~200 world units around selected car. Camera center lerps to car position with look-ahead based on heading direction. Selected via timing tower click (T2.6 event).
4. Onboard cam: tight zoom to ~80 world units around car. Canvas rotated so car heading is always "up". Track and other cars scroll past.
5. Camera state: `{ mode, targetX, targetY, zoom, rotation }` — current values lerp toward target values each frame
6. Smooth transitions: lerp position/zoom/rotation over 0.5 seconds when switching modes
7. Keyboard shortcuts: T = full track, F = follow (requires selected car), O = onboard (requires selected car), ESC = back to full track
8. Camera buttons in controls area
9. All three canvas layers must use the camera transform via `worldToScreen()` from T2.3

# Acceptance Criteria

- [x] Full track view shows entire track (default behavior preserved)
- [x] Follow cam tracks selected car smoothly with look-ahead
- [x] Onboard cam rotates with car heading (car always faces up)
- [x] Smooth transitions between all modes (no jarring jumps)
- [x] Keyboard shortcuts T/F/O/ESC work
- [x] Camera buttons visible in controls
- [x] Track, cars, and overlay all transform correctly with camera
- [x] ESC returns to full track view from any mode
