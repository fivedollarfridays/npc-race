# Objective

Create `tracks.py` with 20 named track presets as control point arrays. Each track approximates a real-world circuit layout, scaled to fit an 800x700 canvas. The file will likely need to be split into multiple modules (tracks/ package) to stay under 400 lines.

# Implementation Plan

1. Create `tracks/` package with `__init__.py` as hub
2. Split tracks into spoke files by character type:
   - `tracks/power.py` — Monza, Baku, Jeddah, Spa (4 tracks)
   - `tracks/technical.py` — Monaco, Singapore, Zandvoort (3 tracks)
   - `tracks/balanced.py` — Silverstone, Suzuka, Austin, Barcelona, Bahrain (5 tracks)
   - `tracks/character.py` — Interlagos, Imola, Melbourne, Montreal, Mugello, Lusail, Hungaroring, Shanghai (8 tracks)
3. Each track: dict with name, country, character, laps_default, control_points
4. `__init__.py` exports merged TRACKS dict + helper functions (get_track, list_tracks, random_track)
5. Write tests for track data validation and helper functions

# Acceptance Criteria

- [ ] 20 named tracks defined with control_points arrays (10-20 points each)
- [ ] Each track has name, country, character, laps_default fields
- [ ] Control points scaled to fit 800x700 canvas (all x in 50-750, y in 50-650)
- [ ] TRACKS dict accessible via `from tracks import TRACKS`
- [ ] `get_track(name)` returns track dict or raises KeyError
- [ ] `list_tracks()` returns sorted list of track keys
- [ ] `random_track()` returns a random track key
- [ ] All files under 400 lines
- [ ] Tests pass for all track data validation

# Verification

- `pytest tests/test_tracks.py -v`
- `bpsai-pair arch check tracks/`
- Verify all 20 tracks render valid splines (no crossing, reasonable shape)
