# T36.2 — Apply Pytest Marks

Classify all 145 test files into 3 tiers using pytest marks.

## Tier definitions

### `@pytest.mark.unit` (or no mark — default)
- No sim calls, no I/O, no subprocess
- Pure function/class tests
- Target: < 0.1s per test

### `@pytest.mark.smoke`
- Runs RaceSim with laps=1, 2-3 cars
- Quick sanity check that sim doesn't crash
- Target: < 5s per test

### `@pytest.mark.integration` (replaces current `@pytest.mark.slow`)
- Full 20-car races, multi-lap, qualifying pipelines
- Target: < 60s per test
- Only ~10-15 tests should have this mark

## Implementation
1. Register marks in `pyproject.toml` or `conftest.py`
2. Migrate existing `@pytest.mark.slow` → `@pytest.mark.integration`
3. Add `@pytest.mark.smoke` to 1-lap sim tests
4. Leave pure unit tests unmarked (default tier)

## AC
- [ ] Marks registered (no warnings)
- [ ] `pytest -m "not smoke and not integration"` runs < 1 min
- [ ] `pytest -m smoke` runs < 3 min
- [ ] `pytest -m integration` runs < 10 min
- [ ] Total test count unchanged
