# T12.2 — Simulation Integration: Filtered Strategy State

## Objective

Wire visibility.py into simulation.py. Replace the current `nearby_cars` with filtered opponent data. Add `opponent_info` list to strategy state with observable-only fields for all cars.

## CONSTRAINT

simulation.py: 372 lines / 15 functions. Target: ≤ 385 lines / 15 functions.

## Implementation Plan

### Import

```python
from .visibility import build_opponent_info, filter_nearby_cars
```

### In build_strategy_state()

1. Replace current `nearby` list with filtered version:
```python
nearby_raw = [{"name": o["name"], "distance_ahead": o["distance"] - cs["distance"],
               "speed": o["speed"], "lateral": o["lateral"],
               "tire_compound": o.get("tire_compound", "medium"),
               "tire_age_laps": o.get("tire_age_laps", 0)}
              for o in self.states if o["car_idx"] != cs["car_idx"]
              and abs(o["distance"] - cs["distance"]) < 100]
```

2. Add `opponent_info` to the return dict:
```python
"opponent_info": build_opponent_info(
    self.states, cs["car_idx"], positions, self.timings,
    self.TICKS_PER_SEC, self.tick),
```

### Backward compatibility

The existing `nearby_cars` field stays but now includes compound and tire_age (observable). No private data leaked. Existing car strategies continue to work unchanged.

### Replay enrichment

No replay changes needed — replay already has all data for post-race analysis (that's the "download" you get after the race but not during).
