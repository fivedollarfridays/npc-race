# Objective

Wire tracks.py into the engine so named track presets can be used instead of procedural generation. Add track selection logic to RaceSim.

# Implementation Plan

1. Add `track_name` parameter to RaceSim / run_race
2. When track_name provided, load control points from tracks.py instead of generating
3. Use track's laps_default if laps not explicitly specified
4. Add track_name to replay JSON output
5. Keep procedural generation as fallback when no track_name given
6. Write tests

# Acceptance Criteria

- [ ] `run_race(track_name="monza")` uses Monza control points
- [ ] Track's laps_default used when laps not specified
- [ ] Explicit --laps overrides track default
- [ ] Procedural generation still works when no track_name
- [ ] `track_name` field included in replay JSON
- [ ] Invalid track_name raises clear error
- [ ] Tests cover named track, procedural fallback, invalid track

# Verification

- `pytest tests/test_engine_tracks.py -v`
- `python play.py --track monza --no-browser` produces valid replay
