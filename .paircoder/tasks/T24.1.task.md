# T24.1 — Watchdog Timeout

Add a 1ms hard timeout to `_safe_call` in `parts_runner.py`. If a player function doesn't return within 1ms, kill it and use the default output. Current implementation only catches exceptions — needs a thread-based watchdog that enforces wall-clock time.

At 30 Hz × 20 cars × 10 parts = 6,000 calls/second. A 5ms function stalls the entire tick. The watchdog prevents this.

Fallback behavior: default output + status="timeout" in the call log.
