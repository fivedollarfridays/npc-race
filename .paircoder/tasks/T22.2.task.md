# T22.2 — Traction Limits

## Problem

Currently nothing limits acceleration or braking by tire grip. A car can apply infinite engine force or brake at any G-force regardless of grip. Real F1: max force = mu × normal_force. This is the traction circle.

## The Fix

```python
def compute_traction_limit(tire_grip_mu, mass_kg, downforce_n):
    """Maximum force the tires can transmit (acceleration OR braking OR cornering)."""
    normal_force = mass_kg * 9.81 + downforce_n
    return tire_grip_mu * normal_force

def apply_traction_limit(requested_force, traction_limit, lateral_force):
    """Apply traction circle: sqrt(longitudinal^2 + lateral^2) <= limit."""
    available_longitudinal = sqrt(max(0, traction_limit^2 - lateral_force^2))
    return min(abs(requested_force), available_longitudinal) * sign(requested_force)
```

This means:
- In a corner (high lateral force), less longitudinal force available → can't accelerate as hard
- On a straight (zero lateral), full traction available → max acceleration
- Tire wear reduces mu → less grip everywhere
- More downforce → higher limit → faster corners AND harder braking

## Player Code Impact

- `differential` lock_pct affects how torque is split → affects traction on corner exit
- `brake_bias` affects which end locks first → poor bias = early lockup = less braking force
- `suspension` ride height affects downforce → affects traction limit everywhere
- `ers_deploy` adds power → can exceed traction limit if deployed in corners
