# T17.2 — Viewer WebSocket Client

## Objective

Add WebSocket streaming mode to `viewer/js/main.js`. When the viewer detects a WS server, it connects and receives frames in real-time instead of loading a giant JSON file.

## Design

**Connection flow:**
1. On page load, try `ws://localhost:8766`
2. If connected, hide "drop replay" message, show "Connected — waiting for race"
3. On `init` message: store track/metadata, init canvas + panels
4. On `frame` message: render the frame immediately (no accumulator needed)
5. On `results` message: show results overlay, store for diagnostics
6. If WS fails, fall back to existing fetch('replay.json') behavior

**Playback controls over WS:**
- Pause/Resume sends `{"type": "pause"}` / `{"type": "resume"}`
- Speed control sends `{"type": "speed", "value": 2}`

## Changes to `viewer/js/main.js`

Add at the top (before fetch):
```javascript
var _ws = null;
var _wsFrames = [];
var _wsMode = false;

function connectWebSocket() {
  try {
    _ws = new WebSocket('ws://localhost:8766');
    _ws.onopen = () => { _wsMode = true; /* show connected */ };
    _ws.onmessage = (event) => {
      var msg = JSON.parse(event.data);
      if (msg.type === 'init') handleWsInit(msg);
      else if (msg.type === 'frame') handleWsFrame(msg);
      else if (msg.type === 'results') handleWsResults(msg);
    };
    _ws.onclose = () => { _wsMode = false; };
    _ws.onerror = () => { /* fall back to fetch */ };
  } catch(e) { /* no WS server, use file mode */ }
}

function handleWsInit(msg) { /* init track, canvases, panels */ }
function handleWsFrame(msg) { /* append frame, render */ }
function handleWsResults(msg) { /* show results, narrative */ }
```
