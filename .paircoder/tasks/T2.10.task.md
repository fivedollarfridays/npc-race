# Objective

Final integration gate: wire all viewer modules together, remove old sidebar, add bottom control bar, build final viewer.html, verify backward compatibility and performance.

# Implementation Plan

1. Remove old sidebar HTML and CSS — go full-width track view with overlay HUD
2. Add bottom bar overlay: playback controls (play/pause, speed buttons, scrubber) + sound controls (volume slider, mute) + camera mode buttons (Track/Follow/Onboard)
3. Wire all JS modules together in correct initialization order in `viewer/viewer.html`
4. Ensure all INJECT markers reference correct files
5. Run `python scripts/build_viewer.py` to produce final `viewer.html`
6. Test backward compat: load old replay.json without new fields — should degrade gracefully (circles if no seg data, no kerbs if no curvatures)
7. Performance check: 60fps sustained with 5 cars on 496-point track

# Acceptance Criteria

- [ ] No sidebar — full-width track view with overlay HUD
- [ ] Bottom bar has playback, sound, and camera controls
- [ ] All features integrated: track renderer, car renderer, overlay, sound, effects, camera
- [ ] `python scripts/build_viewer.py` produces single self-contained viewer.html
- [ ] Old replay files produce degraded but functional view
- [ ] 60fps sustained during playback (check with DevTools)
- [ ] Drag-and-drop replay loading still works
- [ ] Auto-load replay.json from same directory still works

# Verification
