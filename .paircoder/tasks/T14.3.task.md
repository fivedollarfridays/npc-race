# T14.3 — Drama-Driven Crowd + Event Wiring

## Objective

Wire the crowd audio to narrative events. Crowd roar scales with "drama score" — overtakes, battles, safety cars trigger louder crowd reactions.

## Design

**Drama score (0.0-1.0):**
- Base: 0.1 (ambient murmur)
- Overtake: +0.4 for 3 seconds
- Battle (close gap): +0.2 sustained
- Safety car: +0.3 for 5 seconds
- Spin: +0.5 for 2 seconds
- Fastest lap: +0.2 for 2 seconds

`triggerCrowdSwell()` already exists — enhance it to accept a magnitude parameter.

**Event wiring:**
- Replay JSON now has `events` and `commentary` from Sprint 13
- In `main.js`, after loading replay, index events by tick
- During playback, when current tick matches an event, trigger crowd + show commentary overlay

## Changes

### `viewer/js/sound-engine.js`
- Add `_dramaScore` state variable
- Add `updateDramaScore(events, currentTick)` — computes drama level from recent events
- Modify `triggerCrowdSwell(magnitude)` to accept magnitude (0.0-1.0)
- Crowd gain = `_dramaScore * 0.5` (scaled)

### `viewer/js/main.js`
- After loading replay, build `_eventsByTick` index from `replay.events`
- In render loop, check for events at current tick
- Trigger `updateDramaScore()` each frame
- Show commentary text in overlay when event fires

## Tests

- `test_drama_score_function` — JS has `updateDramaScore`
- `test_crowd_swell_magnitude` — `triggerCrowdSwell` accepts parameter
- `test_events_index_in_main` — main.js references `_eventsByTick`
