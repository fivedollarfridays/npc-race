# T34.0b — Fix reliability_score Persistence

`generate_quality_report` computes reliability_score but never stores it
in the car dict. `results.py` reads `car.get("reliability_score", 1.0)`
which always returns 1.0.

## Fix
In `engine/race_runner.py`, after quality report generation, store
`qr.reliability_score` into `car["reliability_score"]`.
