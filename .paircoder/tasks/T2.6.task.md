# Objective

Add F1 TV-style broadcast overlays: timing tower, overtake notifications, lap counter, speed readout, race status badges, and fastest lap indicator.

# Implementation Plan

1. Create `viewer/js/overlay.js` with `renderOverlay(ctx, replay, frame, prevFrame, state)` function
2. Timing tower (left side): vertical stack showing position number, color bar, 3-letter name abbreviation (first 3 chars), gap to leader in seconds (computed from distance delta / leader speed)
3. Overtake notification: detect position changes between frames, show banner at bottom: "OVERTAKE — {name} passes {name} for P{n}", fade out after 2 seconds
4. Lap counter: top center, "LAP X / Y" in clean font
5. Speed readout: leader's speed next to lap counter
6. Race status: "RACE" (green badge during race), "FINAL LAP" (yellow, leader on last lap), "CHEQUERED FLAG" (checkered, leader finishes)
7. Fastest lap: track per-car cumulative lap times, show purple dot next to driver with fastest completed lap in timing tower
8. Click handler on timing tower rows to select car (dispatch event for camera system T2.9)

# Acceptance Criteria

- [x] Timing tower shows all cars with correct positions and gaps
- [x] Gap to leader displayed in seconds (e.g., "+0.3s")
- [x] Overtake events produce visible notification banners
- [x] Lap counter shows correct lap/total
- [x] Race status transitions: RACE -> FINAL LAP -> CHEQUERED FLAG
- [x] Fastest lap indicator appears after first completed lap
- [x] Timing tower rows are clickable (fires selection event)
- [x] Overlay is readable at default zoom and doesn't obscure track

# Verification

- Build viewer, load 2-lap replay, verify timing tower updates
- Watch for overtakes — verify notification appears and fades
- Verify status transitions through race lifecycle
