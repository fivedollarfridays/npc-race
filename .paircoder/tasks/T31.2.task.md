# T31.2 — Call Log Export

The load-bearing task. The viewer can't show code execution without call data in the replay.

PartsRaceSim already has self.call_logs (populated per-tick). This task exports
them into the replay JSON so the viewer can display them.

## Changes
- engine/replay.py: add call_logs to export_replay() output
- Sample every 30th tick (1 per second) to keep file size reasonable
- Per-car format: [{tick, parts: [{name, output, status, efficiency}]}]
- Include glitch info: status="glitch" entries should have glitch_reason

## AC
- Replay JSON includes "call_logs" key
- Sampled at 1Hz (not 30Hz) for file size
- Status values: ok, clamped, glitch, error, timeout
- Existing replay consumers (viewer) handle missing call_logs gracefully
