# T19.5 — Simulation Integration + Strategy API Refactor

## Objective

Wire the driver model into simulation.py. The driver now provides throttle and lateral_target — the strategy() function becomes strategic-only. Backward compatible: if strategy returns throttle, it's used as an override.

## CONSTRAINT

simulation.py: 380 lines / 15 functions. No new functions.

## Implementation Plan

### In __init__

Create one driver per car:
```python
from .driver_model import create_driver
self.drivers = {}
for state in self.states:
    self.drivers[state["name"]] = create_driver(
        track_points, curvatures, distances, headings,
        track_length, {"power": state["power"], "grip": state["grip"], ...})
```

### In _step_car — driver provides base inputs, strategy overrides

```python
# Driver computes optimal inputs
driver = self.drivers[state["name"]]
driver_inputs = compute_driver_inputs(
    driver, state, state["tire_wear"],
    self.weather.get("wetness", 0.0),
    state["damage"]["damage"])

# Strategy provides strategic decisions
# Player CAN override throttle/lateral if they want (advanced mode)
throttle = decision.get("throttle", driver_inputs["throttle"])
lateral_target = decision.get("lateral_target", driver_inputs["lateral_target"])
```

### Strategy API changes

**strategy() return fields — new categories:**

Strategic (the game):
- `pit_request`, `tire_compound_request` — pit strategy
- `engine_mode` — power management
- `ers_deploy_mode` — energy management
- `boost` — one-time boost

Driver overrides (optional, advanced):
- `throttle` — override driver's throttle (default: driver decides)
- `lateral_target` — override driver's line (default: driver decides)

### Update car_template.py

Document the new model: "The driver handles all driving. You handle strategy."
