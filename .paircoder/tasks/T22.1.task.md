# T22.1 — Power-Based Speed Model

## Problem

Current model uses torque × gear ratio → force → acceleration. This produces 305 km/h max instead of 345. The model doesn't naturally produce correct top speeds because it doesn't compute power-at-wheel properly.

## The Fix: Power Model

Real physics: at any speed, available force = power / velocity. Top speed is where force = drag.

```
available_power_watts = engine_hp * 745.7 * torque_pct * mixture_mult + ers_deploy_watts
available_force = available_power_watts / velocity_ms  # F = P/v
drag_force = 0.5 * rho * Cd * A * v^2
net_force = min(available_force, traction_limit) - drag_force - rolling_resistance
acceleration = net_force / mass
```

This naturally produces:
- Top speed where power = drag (no arbitrary cap needed)
- Higher acceleration at low speed (more force available)
- Correct speed-dependent power delivery

## Changes to powertrain_physics.py

Replace torque-based acceleration with power-based:

```python
def compute_power_force(hp, torque_pct, rpm, speed_kmh, ers_deploy_kw, mixture_mult):
    """Compute force available at wheels from power."""
    speed_ms = max(1.0, speed_kmh / 3.6)  # avoid div by zero
    # ICE power: HP * torque_curve(rpm) * torque_pct * mixture
    ice_watts = hp * 745.7 * torque_curve(rpm) * torque_pct * mixture_mult
    # ERS power: direct addition
    ers_watts = ers_deploy_kw * 1000
    total_watts = ice_watts + ers_watts
    # Force = Power / velocity
    return total_watts / speed_ms
```

## Corner Speed Model

Minimum corner speed from physics: `v = sqrt(grip * g * radius)`

Where radius = 1 / curvature. With downforce: `grip = mu * (mass*g + downforce) / (mass*g)`

This gives physically correct minimum speeds for each corner.
