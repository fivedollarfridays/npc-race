# T10.2 — Wet Tire Compounds

## Objective

Add intermediate and wet tire compounds to `engine/tire_model.py`. These compounds have different grip/wear profiles optimized for wet conditions. Update `get_compound_names()` and ensure backward compatibility.

## Design

**New compounds added to COMPOUNDS dict:**

```python
"intermediate": {
    "base_grip": 1.05,        # slightly better than medium base grip
    "wear_rate": 0.000012,    # wears faster than medium
    "cliff_threshold": 0.75,  # cliff comes sooner (softer rubber)
    "cliff_multiplier": 2.5,
    "optimal_temp": 70,       # lower optimal temp (designed for wet cooling)
    "temp_window": 30,
},
"wet": {
    "base_grip": 0.95,        # lower base grip than dry compounds
    "wear_rate": 0.000008,    # very durable
    "cliff_threshold": 0.80,  # late cliff
    "cliff_multiplier": 2.0,
    "optimal_temp": 55,       # very low optimal temp (wet conditions cool tires)
    "temp_window": 35,        # wide temp window (forgiving)
},
```

**Key behaviors:**
- Intermediates: good grip on damp, best at 0.3-0.6 wetness, degrades fast on dry
- Wets: lower base grip but excellent in rain, best at 0.6-1.0 wetness, terrible on dry
- Both have lower optimal temperatures (wet track = cooler tires)
- Existing soft/medium/hard unchanged

**Backward compatibility:**
- `get_compound_names()` returns all 5 names
- `compute_wear()` and `compute_grip_multiplier()` work with new compounds via COMPOUNDS dict lookup
- No changes to existing function signatures

## Changes to `engine/tire_model.py`

1. Add intermediate and wet to COMPOUNDS dict (~15 lines)
2. Update `get_compound_names()` to include new compounds
3. No new functions needed — existing `compute_wear` and `compute_grip_multiplier` already look up from COMPOUNDS
