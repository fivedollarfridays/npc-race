# Objective

Replace single canvas with 3 stacked canvases (background, cars, overlay) and create client-side data enrichment for backward compatibility with old replay files.

# Implementation Plan

1. Update `viewer/viewer.html` — replace single `<canvas id="track">` with three stacked canvases: `trackBg`, `carLayer`, `overlayLayer` using CSS `position: absolute`
2. Create `viewer/js/data-enrichment.js` — on replay load, derive missing fields client-side (headings from track points, curvatures from cross product, seg from nearest-point search) for old replays
3. Refactor `viewer/js/main.js` — separate render pipeline: `renderBackground()` (called once on load/resize), `renderCars()` (per frame), `renderOverlay()` (per frame)
4. Add `worldToScreen(x, y)` transform function that all renderers use, parameterized for camera zoom/pan

# Acceptance Criteria

- [ ] Three canvases render correctly stacked (track underneath, cars middle, overlay on top)
- [ ] Background canvas only redraws on load and resize, not per frame
- [ ] Existing playback controls still work (play/pause, speed, scrubber)
- [ ] Old replay.json files (without track_curvatures/track_headings/seg) still load and render
- [ ] `worldToScreen()` function exists and is used by all renderers
- [ ] No visual regression from current viewer behavior
