# T10.1 — Weather Model

## Objective

Create `engine/weather_model.py` — a weather state machine with probabilistic transitions, grip/wear penalties based on track wetness and tire compound, and a forecast system that can be inaccurate.

## Design

**Weather states (track wetness values):**
```
dry (0.0) → damp (0.2) → wet (0.5) → heavy_rain (0.8) → drying (0.3) → dry (0.0)
```

Transitions are probabilistic per lap. Each lap, roll against transition probabilities:
- `dry → damp`: 8% per lap
- `damp → wet`: 15% per lap
- `damp → dry`: 20% per lap (clears quickly)
- `wet → heavy_rain`: 10% per lap
- `wet → damp`: 15% per lap (drying)
- `heavy_rain → wet`: 20% per lap
- Intermediate states interpolate wetness gradually (not instant)

**Wetness effect on grip (by compound):**
- Dry compounds (soft/medium/hard): `grip *= 1.0 - wetness * 0.6`
  - At wetness 0.5: 70% grip. At 1.0: 40% grip (catastrophic)
- Intermediate: optimal at 0.3-0.6 wetness
  - Penalty outside optimal: `grip *= 1.0 - abs(wetness - 0.45) * 0.8`
  - On dry track: ~64% grip. On soaked track: ~72% grip
- Wet: optimal at 0.6-1.0 wetness
  - Penalty outside optimal: `grip *= 1.0 - max(0, 0.6 - wetness) * 1.0`
  - On dry track: ~40% grip. On damp track: ~80% grip

**Wetness effect on wear:**
- Dry compounds on wet: wear increases (aquaplaning stress)
- Wet compounds on dry: wear increases (overheating)
- Correct compound: wear rate normal

**Forecast:**
- Generate forecast for next 5 laps
- Add noise: predicted_wetness = actual ± random(0.1)
- Forecast updates each lap
