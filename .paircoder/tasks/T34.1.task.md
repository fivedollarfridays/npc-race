# T34.1 — Lap Summary Accumulator

Create `engine/lap_accumulator.py` — collects per-car per-lap data
during simulation without storing full frames.

## API
- `on_tick(states, positions, tick)` — track per-tick state
- `on_lap_complete(car_name, lap, timing)` — finalize lap entry
- `get_lap_summaries() -> dict` — per-car per-lap data
- `get_race_summary() -> dict` — aggregate race data

## Per-lap entry: time, position, compound, tire_wear, pit_stop, fuel

## Acceptance Criteria
- [x] Tracks lap times matching CarTiming.lap_times
- [x] Records pit stops with lap number
- [x] Records tire compound changes
- [x] Records position at each lap end
- [x] Unit tests verify summaries from known tick sequences
