# Objective

Render the track with realistic visuals: asphalt surface, white edge lines, red/white kerbs at corners, run-off areas, checkered start/finish, green grass background, and a faint racing line.

# Implementation Plan

1. Create `viewer/js/track-renderer.js` with `renderTrack(ctx, replay, transform)` function
2. Fill entire background canvas with dark green (grass)
3. Draw run-off area: wider stroke in lighter gray behind the track
4. Draw asphalt surface: fill track path with dark gray, add subtle noise pattern via small repeated canvas texture
5. Draw white edge lines: thin white strokes offset by half track_width from center on both sides using track normals
6. Draw kerbs: at track points where `track_curvatures[i] > threshold`, draw alternating red/white blocks on inside of turn (use heading to determine inside)
7. Draw checkered start/finish: black/white grid pattern across track width at segment 0
8. Draw faint racing line: dotted line through center with slight inside bias at corners

# Acceptance Criteria

- [ ] Track surface has visible asphalt texture (not flat color)
- [ ] White edge lines visible on both sides of track
- [ ] Red/white kerbs appear at high-curvature corners on inside of turns
- [ ] Checkered start/finish pattern clearly visible
- [ ] Green grass surroundings outside track
- [ ] Run-off areas visible between track edges and grass
- [ ] Background renders in under 50ms (performance)
- [ ] Track renders correctly at all zoom levels

# Verification

- Build viewer, load Monza replay, verify all visual elements present
- Load Monaco (tight corners) — verify kerbs appear at multiple corners
- Resize browser window — verify track re-renders correctly
