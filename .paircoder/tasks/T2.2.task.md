# Objective

Create a build system that splits the viewer into modular JS files for development and inlines them into a single self-contained viewer.html for distribution.

# Implementation Plan

1. Create `viewer/` directory with `viewer.html` shell containing CSS and HTML, with `<!-- INJECT:js/filename.js -->` markers replacing the script block
2. Create `viewer/js/main.js` — extract all JS from current viewer.html as-is
3. Create `scripts/build_viewer.py` — reads shell, replaces INJECT markers with file contents wrapped in `<script>` tags, writes to project root `viewer.html`
4. Verify round-trip: built viewer.html works identically to the original

# Acceptance Criteria

- [ ] `python scripts/build_viewer.py` produces `viewer.html` at project root
- [ ] Output is a single self-contained HTML file
- [ ] Output contains `<script>` tags (JS was inlined)
- [ ] Output contains NO `<!-- INJECT:` markers
- [ ] Output contains key functionality strings ("NPC Race", "replay.json", "togglePlay")
- [ ] Existing viewer functionality preserved (drag-and-drop, auto-load, playback)

# Verification

- `python scripts/build_viewer.py`
- `pytest tests/test_build_viewer.py -v`
- `ruff check scripts/build_viewer.py`
