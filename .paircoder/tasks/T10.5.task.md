# T10.5 — Integration Gate: Weather Verification

## Objective

Verify the weather system works end-to-end. Run races and confirm weather transitions occur, grip penalties apply, wet compounds are usable, and races complete normally in all conditions.

## Tests: `tests/test_weather_integration.py`

Mark with `@pytest.mark.slow` since these run full race simulations.

**Class TestWeatherOccurs:**
- `test_weather_can_change` — run 10 races with different seeds, at least 1 should have non-zero wetness in replay frames
- `test_dry_race_completes` — race with weather_enabled=True still completes normally (most races stay dry)
- `test_wetness_in_replay` — replay frames have "track_wetness" field

**Class TestWetGripPenalty:**
- `test_dry_tires_slower_in_wet` — create sim, manually set weather to wet, run a few ticks. Cars on dry tires should have lower speeds than a dry baseline
- `test_intermediate_better_in_damp` — at wetness=0.4, intermediate grip mult > soft grip mult

**Class TestCompoundSwitching:**
- `test_wet_compounds_available` — "intermediate" and "wet" in compound names
- `test_can_pit_for_intermediates` — sim allows pit for intermediate compound

**Class TestArchCompliance:**
- `test_simulation_under_limits` — simulation.py ≤ 400 lines
- `test_simulation_function_count` — ≤ 15 functions
- `test_weather_module_under_limits` — weather_model.py ≤ 140 lines
- `test_tire_model_under_limits` — tire_model.py ≤ 120 lines

Update `scripts/calibration_check.py` to show weather stats:
