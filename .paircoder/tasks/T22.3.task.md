# T22.3 — Sensitivity Measurement

## Objective

Measure the actual lap time impact of each part function by running Monza with default code, then changing ONE part at a time and recording the difference. This tells us whether each part is sensitive enough to reward good code.

## Method

Create `scripts/sensitivity_test.py`:

```python
# 1. Run baseline: all default parts → record lap time
# 2. For each of the 9 codeable parts (not strategy):
#    a. Create an "optimized" version of just that part
#    b. Run with that one part optimized, rest default
#    c. Record lap time difference
# 3. Run with ALL parts optimized → record total gain
# 4. Compare sum of individual gains vs total gain (reveals interactions)
```

## "Optimized" Part Variants

For each part, create a smarter version:

- **engine_map**: RPM-dependent — reduce torque near redline to save engine, maximize in torque band
- **gearbox**: Shift at optimal RPM per gear ratio (not fixed 12000)
- **ers_deploy**: Only deploy on straights above 200 km/h, save for DRS zones
- **ers_harvest**: Max harvest under braking, temperature-aware cutoff
- **brake_bias**: Speed-dependent — more front at high speed, shift rear at low speed
- **suspension**: Speed-adaptive — lowest at 250+ km/h, raise for bumps
- **cooling**: Predictive — minimal cooling until temps approach limits
- **fuel_mix**: Sector-based — rich on straights, lean in corners
- **differential**: Corner-phase-optimized with lateral_g awareness

## Target Sensitivity

Each part should be worth 0.1-0.5s/lap when optimized vs default. Total individual gains should sum to ~2.5-3.5s. Total combined gain should be ~3.0s (showing some interaction — not purely additive).

If a part is worth < 0.05s → its physics isn't sensitive enough, needs tuning.
If a part is worth > 1.0s → too dominant, other parts don't matter enough.

## Output

Print a table:
```
Part            Default    Optimized   Gain
engine_map      1:22.0     1:21.6      0.4s
gearbox         1:22.0     1:21.7      0.3s
...
ALL OPTIMIZED   1:22.0     1:19.2      2.8s
SUM OF PARTS                           3.1s  (interactions: -0.3s)
```

## Tests

- `test_baseline_runs` — default code produces a lap time
- `test_each_part_measurable` — each optimized part produces a different time than baseline
- `test_total_spread_in_range` — all-optimized vs default is 2-4 seconds
- `test_no_single_part_dominant` — no single part > 40% of total spread

## Acceptance Criteria

- Sensitivity table generated for all 9 parts
- Each part contributes measurably (>0.05s)
- No single part dominates (>40% of total)
- Total spread 2-4 seconds
- Interaction effects visible (sum ≠ total)
