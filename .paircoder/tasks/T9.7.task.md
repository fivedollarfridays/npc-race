# T9.7 — Integration Gate: Chaos Verification

## Objective

Verify the drama engine works end-to-end. Run multiple races and confirm that incidents happen, safety cars deploy, damage accumulates, and races are no longer deterministic.

## Tests: `tests/test_drama_integration.py`

**Class TestIncidentsOccur:**
- `test_spins_happen_over_many_races` — run 10 races (3 laps each), at least 1 race should have a spin (in_spin=True in some frame)
- `test_contacts_possible` — run 10 races, verify collision detection runs without errors
- `test_damage_can_accumulate` — verify damage model is wired (damage field exists in frames)

**Class TestSafetyCarIntegration:**
- `test_sc_can_deploy` — run 20 races, at least 1 should have a safety_car=True frame
- `test_sc_limits_speed` — when safety_car=True, no car exceeds 130 km/h in that frame
- `test_sc_compresses_gaps` — after SC phase, gap_ahead_s values should be smaller than before SC

**Class TestRaceVariability:**
- `test_different_seeds_different_results` — run same track/laps with seed=1 and seed=2, results should differ (different finish order or times)
- `test_race_still_completes` — 5-lap monza race finishes (all cars finish or DNF)
- `test_replay_has_all_drama_fields` — frames have damage, in_spin, safety_car

**Class TestArchCompliance:**
- `test_simulation_under_limits` — simulation.py ≤ 395 lines
- `test_simulation_function_count` — ≤ 15 functions
- `test_new_modules_under_limits` — collision.py ≤ 120, damage.py ≤ 100, incident.py ≤ 100, safety_car.py ≤ 140

## Calibration script update

Update `scripts/calibration_check.py` to show incident stats:
- Number of spins per race
- Number of contacts per race
- Safety car deployments per race
- DNFs per race

## Acceptance Criteria

- ≥ 1320 tests passing

- Race variability: different seeds produce different outcomes
