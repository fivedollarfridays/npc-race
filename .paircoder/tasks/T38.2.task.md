# T38.2 — CLI Error Handling

## Bug 4: Invalid track proceeds on random
In `cli/commands.py:cmd_run`, when `_resolve_track` returns None
and `args.track` was specified, abort instead of continuing.

## Bug 5: Error exits return 0
All `cmd_*` functions that print errors must `sys.exit(1)` or
return 1. Check: cmd_init, cmd_run, cmd_submit, cmd_validate.
The CLI `main()` must propagate return codes.

## Bug 6: Traceback on bad car-dir subdir
`npcrace run --car-dir cars/audit-car` crashes with traceback.
Add try/except in cmd_run around run_race, print friendly error.
