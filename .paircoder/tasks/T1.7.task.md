# Objective

Create a proper CLI package so users can run `npcrace run`, `npcrace init`, `npcrace validate`, etc. Uses pyproject.toml entry points. No external dependencies.

# Implementation Plan

1. Create `cli/` package:
   - `cli/__init__.py`
   - `cli/main.py` -- top-level CLI dispatcher (argparse subcommands)
   - `cli/commands.py` -- init, wizard, validate, run command implementations
2. Create `pyproject.toml` with `[project.scripts]` entry point: `npcrace = "cli.main:main"`
3. Commands:
   - `npcrace run` -- same as current play.py (--car-dir, --laps, --seed, --track, --output, --no-browser)
   - `npcrace init` -- create cars/ directory with template car file
   - `npcrace validate [car_file]` -- run bot_scanner on car file(s)
   - `npcrace wizard` -- interactive car creation wizard (asks name, color, stat allocation)
   - `npcrace list-tracks` -- print available tracks
4. Keep play.py working as standalone script
5. Write tests

# Acceptance Criteria

- [ ] pyproject.toml defines npcrace entry point
- [ ] `npcrace run --track monza` works after pip install -e .
- [ ] `npcrace init` creates cars/ with template
- [ ] `npcrace validate cars/gooseloose.py` runs scanner
- [ ] `npcrace wizard` walks through car creation interactively
- [ ] `npcrace list-tracks` prints track table
- [ ] play.py still works standalone
- [ ] All files under 400 lines
- [ ] Tests cover each subcommand
