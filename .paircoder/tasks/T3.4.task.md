# Objective

Make lateral movement functional in simulation. Strategy returns lateral_target, car lerps toward it, proximity resistance prevents overlap.

# Implementation Plan

1. Add _apply_lateral(state, lateral_target, dt) to simulation.py
2. Lateral change rate = base_rate * (1 - speed/max_speed * 0.5) — faster = less agile
3. state["lateral"] lerps toward lateral_target at rate * dt
4. Clamp to [-1, 1]
5. Proximity resistance: if another car within 10 distance units and similar lateral (within 0.3), push apart
6. Extract lateral_target from strategy decision with default 0.0

# Acceptance Criteria

- [x] lateral_target from strategy is respected (car moves toward target)
- [x] Lateral change is not instant (lerp, not snap)
- [x] High-speed cars change lateral slower than low-speed cars
- [x] Two close cars resist same lateral position
- [x] Default lateral_target=0.0 works for old strategies
- [x] simulation.py stays ≤15 functions
