# T14.1 — Spatial Audio

## Objective

Upgrade `viewer/js/sound-engine.js` from single-car leader-focused audio to multi-car spatial audio. Each car gets an engine oscillator with distance-based volume relative to the camera position.

## Design

**Current state:** One sawtooth oscillator tracks the leader's speed. All other cars are silent.

**Target:** Up to 5 car audio slots. Each car gets:
- Engine oscillator (pitch = speed-mapped frequency)
- Volume = `1.0 / (1.0 + distance_to_camera * 0.01)` (inverse distance)
- Pan = left/right based on screen position

**Implementation:**
- Create `_carAudioSlots` array with oscillator + gain per car
- `updateSpatialAudio(cars, cameraPos)` updates each slot's frequency, gain, pan
- Limit to nearest 5 cars to camera to save resources
- Fade in/out when cars enter/leave audio range

## Changes to `viewer/js/sound-engine.js`

1. Add `_carAudioSlots = []` state
2. Add `initSpatialAudio(numCars)` — create oscillator+gain+panner per car
3. Modify `updateSound(replay, frameIdx)` to call new spatial update
4. Keep backward compat — if no camera data, fall back to leader-only mode

## Tests (in `tests/test_sound_engine.py`)
