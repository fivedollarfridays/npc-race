# T19.4 — Reactive Driver

## Objective

Add reactive behavior to `engine/driver_model.py` — the driver adapts when conditions aren't ideal. Uses the existing learning system (cars/data/*.json) to improve recovery behavior over multiple races.

## Design

**Reactive triggers:**
1. **Post-collision recovery** — just got bumped, speed dropped unexpectedly
   - Conservative: lift to 60% throttle for 30 ticks, regain composure
   - Learned: if data shows recovery was successful at 80%, try that next time
2. **Overtake opportunity** — car ahead is slower through a section
   - Conservative: wait for DRS zone
   - Learned: if passing outside at Turn X worked before, try it again
3. **Tire cliff management** — wear > 0.7, grip degrading fast
   - Conservative: reduce speed profile by 20%
   - Learned: if data shows 15% reduction was enough, use that
4. **Wet transition** — track changing from dry to wet
   - Conservative: immediately reduce 30%
   - Learned: actual wet grip data from previous races

**Learning storage:**
```json
{
  "recovery_throttle": 0.75,     // learned from collision events
  "overtake_aggression": 0.5,    // 0=wait for DRS, 1=dive every corner
  "tire_cliff_reduction": 0.15,  // how much to back off near cliff
  "wet_caution": 0.25            // speed reduction in wet
}
```

Starts at conservative defaults. After each race, adjusts based on outcomes (did the recovery work? did the overtake succeed?).
