# Objective

Add track geometry data and car segment index to replay.json so the viewer can render realistic track details and car orientation.

# Implementation Plan

1. Add `compute_track_headings(track_points)` to `engine/track_gen.py` — returns list of heading angles (radians) per track point using atan2 of direction vector to next point
2. In `engine/replay.py:record_frame()`, include `"seg": seg` in each car's frame dict (seg is already computed by `get_track_pos()` but discarded)
3. Update `engine/replay.py:export_replay()` to accept and include `track_curvatures` and `track_headings` arrays (rounded to 4 decimal places)
4. Thread curvatures and headings through `engine/simulation.py` to the export call

# Acceptance Criteria

- [ ] `replay.json` contains `track_curvatures` (array of floats, length == len(track))
- [ ] `replay.json` contains `track_headings` (array of floats in radians, length == len(track))
- [ ] Each car in each frame contains `seg` (integer in range [0, len(track)))
- [ ] `compute_track_headings` returns correct values for a known shape
- [ ] Existing tests still pass
- [ ] New tests verify all three new fields
