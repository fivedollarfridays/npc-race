# T21.3 — Wire Parts Runner Into Simulation

The big one. Replace the current `_step_car` physics (old stat-based throttle/speed computation) with the parts runner. Each tick, `_step_car` calls `run_parts_tick()` which calls all 10 player part functions, feeds them through the 3 physics engines, and updates car state.

simulation.py is at 397/400 lines, 15/15 functions. This is a REPLACEMENT not an addition — the old physics code in `_apply_physics`, `_apply_tire_wear`, etc. gets replaced by parts runner calls. Net line count should decrease.

The driver model (speed_profile + racing_line) provides throttle_demand and lateral to the parts runner as physics_state inputs. Strategy function still called for pit/mode decisions.

Part call logs stored on sim for WebSocket streaming to the live terminal UI (Sprint 22).
