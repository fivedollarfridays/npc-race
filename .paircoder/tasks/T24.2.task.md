# T24.2 — Efficiency Framework

Build the framework for computing per-part efficiency factors against t-1 state.

1. At the start of each tick, snapshot the previous tick's state (speed, temps, ride_height, cooling, battery)
2. After each part call, compute its efficiency: actual_result / optimal_result_for_this_situation
3. Store efficiencies in the tick's call log
4. At end of tick, multiply all efficiencies → realized_performance_fraction
5. Apply fraction to the theoretical max speed change for this tick

The "optimal result" for each part is computed by the physics — not hardcoded. It's what the physics says would produce the best outcome given t-1 conditions.
