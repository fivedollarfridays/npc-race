# T8.4 — Car Telemetry Panel JS

## Objective

Build the right-column telemetry panel: deep readouts for the selected car. This is where the feedback loop lives — the player sees exactly what's happening to their car and why.

## New File: `viewer/js/telemetry-panel.js` (~200 LOC)

### Live Readouts Section

Monospaced, right-aligned numbers. Updated every frame.

```
Speed     287 km/h
Tire      ██████░░ 62%  MEDIUM (12 laps)
Temp       94°C  [OPTIMAL]
Fuel      ███████░ 71%
DRS       READY
Engine    PUSH
Dirty Air ▓▓░ 0.94
Gap ▲     +1.203s
Gap ▼     -0.892s
Pit Stops 1
```

Each readout is a `<div class="readout">` with label + value spans.

**Color rules:**
- Speed: white (just a number)
- Tire wear bar: green (<50%), yellow (50-75%), red (>75%)
- Temp: green (in optimal window), yellow (close to edge), red (outside or near cliff)
- Fuel bar: blue
- DRS: green when ACTIVE, white when READY, gray when UNAVAIL
- Engine: red=PUSH, white=STANDARD, blue=CONSERVE
- Dirty air: orange bar when < 1.0, gray when clean
- Gap: green (gaining = value shrinking vs prev frame), yellow (losing)

### Sector Comparison Table

Shows current lap sectors vs personal best:

```
        Current    Best     Delta
S1      32.112    31.890    +0.222  ← yellow
S2      31.456    31.890    -0.434  ← purple (session best!)
S3      ...running...
Lap     63.568    63.780    -0.212  ← green
```

Built from `current_sector`, `last_sector_time`, `last_sector_idx`, and `best_lap_s` in frame data. Client tracks sector times per lap as they arrive.

**Colors:** Purple = session best across all cars. Green = personal best. Yellow = slower.

### "Aha Moment" Alerts

Queue of max 3 alerts, newest at top, fade out after 5 seconds:

```
⚠ DIRTY AIR: -0.08 grip in sector 2
⚠ TIRE CLIFF: Wear 0.78 → cliff at 0.80!
✓ UNDERCUT: Pit gained +1 position
```

**Trigger conditions (computed client-side):**
- Dirty air: `in_dirty_air` transitions from false → true, show factor loss
- Tire cliff: `tire_wear > 0.75` (or compound-specific cliff threshold)
- Position change: position improves after pit stop → "UNDERCUT: gained +N"
- Position change: position worsens after pit stop → "OVERCUT: lost -N"
- Fuel warning: fuel_pct < 0.10 → "FUEL CRITICAL"
