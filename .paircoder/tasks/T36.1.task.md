# T36.1 — Pin All Sim-Calling Tests to Explicit Laps

The root cause: tests that call `RaceSim()` or `run_race()` without
specifying `laps=` inherit track defaults (now 53-78 laps). Every sim
test must specify `laps=1` or `laps=2` explicitly.

## Scope (46 test files, ~462 real sim tests)

Find every call to `RaceSim(`, `run_race(`, `run_qualifying(`, and
`main(["run"` across tests/. For each:
1. If `laps=` is missing → add `laps=1` (or `laps=2` if multi-lap behavior is under test)
2. If `--laps` is missing from CLI calls → add `--laps 1`
3. Never rely on track defaults in tests

## Do NOT change
- The actual track defaults (those are correct at real F1 values)
- Test logic or assertions
- Production code

## AC
- [ ] `grep -r "RaceSim(" tests/ | grep -v "laps="` returns zero matches
- [ ] `grep -r "run_race(" tests/ | grep -v "laps="` returns zero matches
- [ ] `grep -r 'main(\["run"' tests/ | grep -v "laps"` returns zero matches
- [ ] All tests pass
- [ ] No test takes > 30s individually (verify with --timeout=30)
