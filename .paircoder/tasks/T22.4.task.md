# T22.4 — Interaction Calibration

## Objective

Based on T22.3 sensitivity results, tune the physics so part interactions are realistic. If the sensitivity test reveals issues (a part is too weak, too dominant, or doesn't interact), adjust the physics coefficients.

## Known Interactions to Verify

1. **brake_bias × ers_harvest**: Aggressive harvest changes rear brake feel. If player doesn't account for this in brake_bias, car becomes unstable under braking.

2. **suspension × cooling**: Low ride height increases downforce but reduces cooling airflow. Player must balance these.

3. **engine_map × fuel_mix**: Rich mixture amplifies engine_map torque. If both are aggressive simultaneously, fuel runs out.

4. **ers_deploy × differential**: Deploying ERS in corners adds rear torque. If diff is locked, this causes understeer. If open, wheelspin.

5. **gearbox × engine_map**: Wrong gear for the engine map's optimal RPM range = power loss.

## Method

For each interaction pair, test:
- Part A optimized + Part B default → gain_A
- Part A default + Part B optimized → gain_B
- Part A optimized + Part B optimized → gain_AB
- If gain_AB ≈ gain_A + gain_B → no interaction (bad, parts are independent)
- If gain_AB < gain_A + gain_B → negative interaction (parts conflict)
- If gain_AB > gain_A + gain_B → synergy (parts amplify each other)

Real F1 has all three: some synergies, some conflicts.

## Tuning

If interactions are too weak: increase the coupling coefficients in physics (e.g., make cooling affect drag more, make ERS harvest affect brake balance more).

If interactions are too strong: reduce coupling so individual parts still matter on their own.

## Acceptance Criteria

- At least 3 interaction pairs show measurable coupling (gain_AB ≠ gain_A + gain_B)
- No interaction is catastrophic (combining two optimized parts shouldn't be WORSE than default)
- Document the interaction matrix for players
